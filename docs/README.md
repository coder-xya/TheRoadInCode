## 📖 文档使用指南

### 文档层级

```
docs/
├── 00_learning/          # 思维层：为什么这样设计
├── 01_准备工作/          # 规划层：做什么
├── 02_架构详解/          # 理解层：怎么工作
└── 03_开发任务/          # 执行层：怎么做（当前）
```

### 更新原则

1. **需求变化** → 更新 `01_准备工作/requirements.md`
2. **架构调整** → 更新 `02_架构详解/` 对应文档
3. **任务变更** → 更新 `03_开发任务/` 里程碑文档
4. **完成里程碑** → 更新进度状态，创建下一个任务文档

## 🔍 常见问题

### Q: 文档太多，不知道从哪里开始？

**A**: 直接看 [M1 任务文档](./03_开发任务/M1-项目初始化与Entry基座.md)，从任务 1.1 开始实施

### Q: 想了解某个技术的详细用法？

**A**: 查阅 `02_架构详解/` 对应的技术文档

### Q: 需求变了，要改哪些文档？

**A**:

1. 更新 `01_准备工作/requirements.md`（功能需求）
2. 如果影响架构，更新 `02_架构详解/12_后端架构与API契约.md`
3. 调整 `03_开发任务/` 当前里程碑的任务清单

### Q: 文档和代码不一致了怎么办？

**A**: 优先以代码为准，及时更新文档；重大变更应该先讨论、更新文档、再改代码

## 🤝 协作方式

### 开发流程

1. 查看当前里程碑任务文档
2. 选择一个任务开始实现
3. 完成后更新任务进度（在任务文档中勾选）
4. 遇到问题查阅技术文档或提问
5. 里程碑完成后创建下一个里程碑文档
